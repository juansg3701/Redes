Instalación

Anterior Siguiente

Se descarga windows server 2012 r2, se configura. Como pasos iniciales se debe colocar una dirección IP fija, desactivar el firewall y tener la máquina virtual como host-only, además de no tener activado el DHCP por defecto en el software de virtualización

DNS

Anterior Siguiente

Para instalar el sistema de nombres de dominio (DNS), se realiza por el panel de administración del servidor, luego se crea una nueva zona y un host para esa zona. También se puede descargar servicios de dominio active directory y se configura la nueva zona. Se prueba con ping y el dominio creado.

DHCP

Anterior Siguiente

Para el Protocolo de configuración dinámica de host (DHCP), se descarga también del administrador de servidor y se configura creando un nuevo ámbito, otorgando el intervalo para asignar las IP, se prueba cuando se han dado concesiones a otros equipos.

FTP

Anterior Siguiente

Para el protocolo de transferencia de archivos se necesita instalar Internet Information Services (IIS), luego se crea una carpeta que podrán acceder todos los clientes de la red. Se prueba desde un cliente al cargar y descargar archivos.

SMTP

Anterior Siguiente

Para el protocolo para transferencia simple de correo, se puede descargar mailEnable. Se debe tener cuidado con las configuraciones para acceder localmente y externamente dentro de la red. Se prueba enviando correos de un usuario a otro, dentro de la misma red pero con diferente equipo.

HTTP

Anterior Siguiente

El protocolo de transferencia de hipertexto se descarga e instala junto con el FTP en el paquete del IIS, por lo tanto solo se debe configurar un nuevo sitio para probar su correcto funcionamiento.
